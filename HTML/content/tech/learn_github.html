Guide to Checking In Code to GitHub
GitHub मा कोड चेक-इन गर्ने गाइड

Prerequisites
पूर्वावश्यकताहरू

  • Git installed on your computer
    तपाईंको कम्प्युटरमा Git इन्स्टल गरिएको
  • GitHub account
    GitHub खाता
  • Repository created on GitHub
    GitHub मा रिपोजिटरी सिर्जना गरिएको
  • Git configured with your credentials
    तपाईंको परिचयपत्रहरूसँग Git कन्फिगर गरिएको
First-time setup? Configure Git with these commands:
पहिलो पटक सेटअप? यी कमाण्डहरूसँग Git कन्फिगर गर्नुहोस्:
git config --global user.name "Your Name" git config --global user.email "your.email@example.com"

Basic Workflow
आधारभूत कार्यप्रवाह

1. Clone the Repository (First Time Only)
१. रिपोजिटरी क्लोन गर्नुहोस् (पहिलो पटक मात्र)

git clone https://github.com/username/repository.git cd repository

2. Create/Switch to a Branch
२. ब्रान्च सिर्जना गर्नुहोस्/स्विच गर्नुहोस्

git checkout -b feature-branch-name

3. Check Status of Your Changes
३. तपाईंको परिवर्तनहरूको स्थिति जाँच गर्नुहोस्

git status

4. Stage Your Changes
४. तपाईंको परिवर्तनहरू स्टेज गर्नुहोस्

# Stage specific files / विशेष फाइलहरू स्टेज गर्न git add filename.ext # Stage all changes / सबै परिवर्तनहरू स्टेज गर्न git add .

5. Commit Your Changes
५. तपाईंको परिवर्तनहरू कमिट गर्नुहोस्

git commit -m "Brief description of your changes"

6. Push to GitHub
६. GitHub मा पुश गर्नुहोस्

# First time pushing the branch / पहिलो पटक ब्रान्च पुश गर्दा git push -u origin feature-branch-name # Subsequent pushes / त्यसपछिका पुशहरू git push

Best Practices
उत्तम अभ्यासहरू

Commit Message Guidelines
कमिट सन्देश दिशानिर्देशहरू

  • Use clear, descriptive commit messages
    स्पष्ट, वर्णनात्मक कमिट सन्देशहरू प्रयोग गर्नुहोस्
  • Start with a verb (Add, Fix, Update, Refactor)
    क्रिया बाट सुरु गर्नुहोस् (थप्नुहोस्, सुधार्नुहोस्, अपडेट गर्नुहोस्, रिफ्याक्टर गर्नुहोस्)
  • Keep the first line under 50 characters
    पहिलो लाइन ५० अक्षरभन्दा कम राख्नुहोस्

Branch Management
ब्रान्च व्यवस्थापन

  • Keep branches focused on single features/fixes
    ब्रान्चहरूलाई एकल फिचर/फिक्सहरूमा केन्द्रित राख्नुहोस्
  • Regularly sync with main branch
    नियमित रूपमा मुख्य ब्रान्चसँग सिंक गर्नुहोस्
  • Delete branches after merging
    मर्ज गरेपछि ब्रान्चहरू मेटाउनुहोस्

Common Scenarios
सामान्य परिदृश्यहरू

Updating Your Branch with Main
तपाईंको ब्रान्चलाई मुख्य ब्रान्चसँग अपडेट गर्दै

git checkout main git pull git checkout your-branch git merge main
