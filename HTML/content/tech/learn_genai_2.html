Introduction to Prompt Engineering

Prompt engineering is the art and science of designing effective inputs for language models to obtain desired outputs. Good prompt engineering can significantly improve the quality, relevance, and accuracy of LLM responses.

Key Principle: The quality of the output is directly proportional to the quality of the input prompt.

प्रम्प्ट इन्जिनियरिङको परिचय

प्रम्प्ट इन्जिनियरिङ भाषा मोडेलहरूको लागि इच्छित आउटपुट प्राप्त गर्न प्रभावकारी इनपुट डिजाइन गर्ने कला र विज्ञान हो। राम्रो प्रम्प्ट इन्जिनियरिङले एलएलएम प्रतिक्रियाहरूको गुणस्तर, सान्दर्भिकता र सटीकता उल्लेखनीय रूपमा सुधार गर्न सक्छ।

मुख्य सिद्धान्त: आउटपुटको गुणस्तर इनपुट प्रम्प्टको गुणस्तरसँग प्रत्यक्ष समानुपातिक हुन्छ।

Core LLM Parameters

Temperature

Range: 0.0 to 1.0

Controls the randomness in the model's responses. Lower values make the output more deterministic and focused, while higher values make it more creative and diverse.

Use cases:
  • 0.0 - 0.3: Factual responses, coding, technical writing
  • 0.4 - 0.7: Balanced creativity and coherence
  • 0.8 - 1.0: Creative writing, brainstorming, poetry

Top-k Sampling

Range: 1 to vocabulary size

Limits the cumulative probability distribution to the k most likely next tokens. Helps prevent the generation of low-probability or nonsensical tokens.

Example settings:
  • k=50: More focused and conservative outputs
  • k=100: Balanced diversity
  • k=500: More varied outputs

Top-p (Nucleus) Sampling

Range: 0.0 to 1.0

Considers the smallest set of tokens whose cumulative probability exceeds the probability p. Also known as nucleus sampling.

Recommended values:
  • 0.1 - 0.3: Very focused, deterministic outputs
  • 0.4 - 0.7: Balanced outputs
  • 0.8 - 1.0: More diverse and creative outputs

मुख्य एलएलएम प्यारामिटरहरू

तापमान (Temperature)

सीमा: 0.0 देखि 1.0

मोडेलको प्रतिक्रियाहरूमा यादृच्छिकता नियन्त्रण गर्दछ। कम मानहरूले आउटपुटलाई बढी निश्चित र केन्द्रित बनाउँछ, जबकि उच्च मानहरूले यसलाई बढी सिर्जनात्मक र विविध बनाउँछ।

टप-के सैम्पलिङ (Top-k)

सीमा: 1 देखि शब्दावली आकार

सबैभन्दा सम्भावित अर्को टोकनहरूको क मा संचयी सम्भावना वितरण सीमित गर्दछ।

टप-पी सैम्पलिङ (Top-p)

सीमा: 0.0 देखि 1.0

टोकनहरूको सबैभन्दा सानो सेट विचार गर्दछ जसको संचयी सम्भावना सम्भावना p भन्दा बढी हुन्छ।

Max Tokens

Limits the length of the generated response. Important for controlling costs and ensuring concise outputs.

Tip: Set this value based on your expected output length, but include some buffer for unexpected verbosity.

Prompt Engineering Techniques

1. Zero-shot Prompting

Asking the model to perform a task without any examples.

Example: 
Classify the following text as positive or negative:
"This product exceeded my expectations!"

2. Few-shot Prompting

Providing a few examples before the actual task.

Example: 
Classify the sentiment (positive/negative):

Text: "This movie was terrible"
Sentiment: negative

Text: "Great service!"
Sentiment: positive

Text: "The food was cold"
Sentiment: [wait for model response]

3. Chain-of-Thought Prompting

Encouraging the model to break down complex problems into steps.

Example: 
Let's solve this step by step:
1) First, understand the problem
2) Break it down into components
3) Solve each component
4) Combine the solutions

३. चेन-अफ-थट प्रम्प्टिङ

जटिल समस्याहरूलाई चरणहरूमा विभाजन गर्न मोडेललाई प्रोत्साहित गर्ने।

Best Practices

  • Be specific and clear in your instructions
  • Provide context and constraints
  • Use appropriate formatting and structure
  • Start with zero-shot, then add examples if needed
  • Iterate and refine based on results

उत्तम अभ्यासहरू

  • निर्देशनहरूमा विशिष्ट र स्पष्ट हुनुहोस्
  • सन्दर्भ र सीमाहरू प्रदान गर्नुहोस्
  • उपयुक्त ढाँचा र संरचना प्रयोग गर्नुहोस्
  • जिरो-शटबाट सुरु गर्नुहोस्, त्यसपछि आवश्यक परेमा उदाहरणहरू थप्नुहोस्
  • नतिजाहरूको आधारमा परिमार्जन गर्नुहोस्

Parameter Combinations

Different tasks require different parameter combinations:

Creative Writing

  • Temperature: 0.8
  • Top-p: 0.9
  • Top-k: 100

Technical Writing

  • Temperature: 0.2
  • Top-p: 0.1
  • Top-k: 40

Balanced Conversation

  • Temperature: 0.5
  • Top-p: 0.5
  • Top-k: 50

Common Pitfalls

  • Using overly high temperature for factual tasks
  • Not providing enough context
  • Inconsistent formatting in few-shot examples
  • Unclear or ambiguous instructions
  • Not setting appropriate max tokens

Advanced Techniques

System Messages

Using system messages to set context and behavior:
